# GARCH-X 모델 성능 평가 지표 및 채택 근거

---

## 1. GARCH 계열 모델을 평가할 때 봐야 할 지표

GARCH 모델은 일반 회귀 모델과 평가 방식이 근본적으로 다릅니다.  
종속변수가 수익률의 **수준(level)**이 아니라 **분산(variance)**이기 때문입니다.

### 1.1 모델 적합성 지표

| 지표 | 개념 | 판단 기준 |
|---|---|---|
| **AIC (Akaike Information Criterion)** | 로그우도 + 파라미터 수 패널티. 과적합 방지. | 낮을수록 좋음. 모델 간 비교에 사용 |
| **BIC (Bayesian Information Criterion)** | AIC보다 파라미터 패널티가 강함 | 낮을수록 좋음. 소표본에서 AIC보다 보수적 |
| **Log-Likelihood** | 데이터가 해당 모델에서 생성될 우도의 로그값 | 높을수록 좋음. AIC/BIC 계산의 기반 |

> 이 분석에서의 실측값:
> ```
> GARCH(1,1)-t : AIC = 38,766.1  BIC = 38,801.7  Log-L = -19,378.0
> EGARCH(1,1)-t: AIC = 38,736.8  BIC = 38,772.4  Log-L = -19,363.4
> ```
> EGARCH가 AIC/BIC 모두 낮아 적합도는 우수하나,
> 분산 방정식의 로그 변환 구조 때문에 GPR을 수동으로 투입하기 어려워 GARCH를 채택.

---

### 1.2 파라미터 유의성 지표

| 지표 | 개념 | 판단 기준 |
|---|---|---|
| **t-stat / p-value** | 각 파라미터가 0과 유의하게 다른지 검정 | p < 0.05 이면 유의 |
| **alpha + beta < 1** | 분산의 안정성(stationary) 조건 | 합이 1 이상이면 분산이 발산함 |
| **nu (자유도)** | t분포의 꼬리 두께. 낮을수록 극단값이 자주 발생 | 2 초과 필수. 보통 4~10이면 현실적 |

> 이 분석에서의 실측값:
> ```
> omega : 0.0569  (p < 0.001, 유의)
> alpha : 0.0752  (p < 0.001, 유의)  ← 충격 지속성
> beta  : 0.9164  (p < 0.001, 유의)  ← 변동성 관성
> nu    : 6.37    (p < 0.001, 유의)  ← 두꺼운 꼬리 확인
> alpha + beta = 0.9916 < 1          ← 안정성 조건 충족
> ```

---

### 1.3 외생변수(GPR) 유효성 지표

| 지표 | 개념 | 판단 기준 |
|---|---|---|
| **delta 유의성** | GPR이 분산에 통계적으로 기여하는지 | p < 0.05 이면 유의 |
| **GPR → 밴드 폭 단조성** | GPR 증가 시 밴드 폭이 실제로 넓어지는지 | 단조 증가이면 설계 의도 달성 |
| **Granger 인과 검정** | GPR이 수익률 방향에 선행하는지 | 이 모델에서는 평균이 아닌 분산 방정식 타겟 |

> 이 분석에서의 실측값:
> ```
> delta : 0.000081  (t = 3.16, p = 0.0016)  ← 유의
> GPR_THREAT → 수익률 방향: p = 0.59       ← 유의하지 않음 (분산 채널이 맞음)
> GPR=80  → 42일 밴드폭 $25.3
> GPR=705 → 42일 밴드폭 $26.3              ← 단조 증가 확인 (v9 기준)
> ```

### 1.4 시뮬레이션 성능 지표 (Monte Carlo 고유)

| 지표 | 개념 | 판단 기준 |
|---|---|---|
| **Coverage Rate (포함률)** | 실제값이 예측 밴드 내에 속한 비율 | Q10~Q90 밴드 기준 약 80% 목표 |
| **Escape Rate (이탈률)** | 실제값이 밴드 밖으로 이탈한 비율 | < 20% 목표 |
| **Interval Sharpness** | 밴드가 불필요하게 넓지 않은지 | Coverage 충족 하에서 좁을수록 좋음 |

> 이 분석에서의 실측값 (v9):
> ```
> Coverage — 1~10일: 50%  /  21~42일: 33~67%
> 이란전쟁 이탈률 — Early(1~10일): 100%  /  Stable(11~42일): 100%
> 러-우 전쟁     — Early: 90%  /  Stable: 25%
> 이스라엘-하마스  — Early: 10%  /  Stable: 21.9%
> ```
> *이란전쟁은 학습 기간 외 신규 이벤트(레짐 변화)로, 이탈은 모델 실패가 아닌 구조적 한계*

### 1.5 분포 진단 지표

| 지표 | 개념 | 판단 기준 |
|---|---|---|
| **첨도 (Kurtosis)** | 수익률 분포의 꼬리 두께 | 44.38 → 정규분포(=0) 대비 극단적 두꺼운 꼬리 |
| **왜도 (Skewness)** | 분포의 좌우 비대칭 | -0.033(lam) → 미세한 하방 편향 |
| **ADF 단위근 검정** | 시계열의 정상성 확인 | 수준값 p=0.25(비정상) / 수익률 p=0.00(정상) |
| **절대수익률 자기상관** | 변동성 군집 존재 확인 | lag 1~10 모두 +0.19~+0.27 → 군집 명확 |

## 2. 이 모델을 채택하게 된 경위

### 2.1 시행착오 요약

```
1차 시도: QuantReg + LightGBM (유가 수준값 예측)
  → 실패 이유: Brent_lag1이 t=3,527로 설명력 독점 (Pseudo R²=0.9713)
               GPR_THREAT 계수 부호 역전 (VIF=37.6, 다중공선성)
               밴드가 실제값을 그대로 추적 → 예측 구간으로 무의미

2차 시도: Multi-horizon LightGBM (h일 후 유가 예측)
  → 실패 이유: dropna로 이란전쟁 시점 데이터 소실
               (fe 마지막 날짜 2026-01-22, 실제 전쟁 2026-02-28)
               Test 셋 15행 → Early stopping 작동 불가
               SHAP: Brent_roll_mean_63이 밴드 폭 결정 (GPR의 100배 기여)
               → LightGBM 구조 자체가 이 분석 목적에 부적합

최종 선택: GARCH-X Monte Carlo
```

### 2.2 GARCH-X를 결정적으로 채택한 이유 3가지

#### ① GPR을 분산 방정식에 직접 투입할 수 있는 유일한 구조

이 분석의 핵심 목적은 **"GPR이 높을수록 밴드가 넓어지는 것"**입니다.  
LightGBM은 어떤 피처든 예측력이 높은 순서로 사용하므로,  
Brent lag가 존재하는 한 GPR이 밴드를 넓히는 것은 구조적으로 불가능합니다.

GARCH-X의 분산 방정식:
```
σ²_t = ω + α·ε²_{t-1} + β·σ²_{t-1} + δ·GPR_{t-1}
                                        ↑
                   GPR이 높으면 내일 변동성이 수학적으로 커짐
```

delta = 0.000081 (p=0.0016) — GPR이 분산에 통계적으로 유의하게 기여함을 확인.

---

#### ② 단위근 문제를 구조적으로 해결

유가 수준값은 단위근(ADF p=0.25, 비정상)입니다.  
수준값을 종속변수로 쓰면 어떤 lag든 설명력을 독점하게 됩니다.  
GARCH-X는 **수익률(pct_change)**을 모델링하므로 이 문제가 발생하지 않습니다.

```
Brent 수준값: ADF p = 0.2505 → 비정상(단위근)
Brent 수익률: ADF p = 0.0000 → 정상
```

---

#### ③ 시뮬레이션 기반이라 데이터 소실 없이 현재 시점 예측 가능

LightGBM은 `Brent_fwd_42` 생성 시 마지막 42행이 dropna로 제거됩니다.  
이로 인해 이란전쟁 발발일(2026-02-28) 데이터가 사라집니다.  
GARCH-X는 현재 시점의 파라미터(GPR, VIX, 기준가)를 입력받아  
미래 경로를 Monte Carlo로 직접 생성하므로 이 문제가 없습니다.

### 2.3 v9에서 추가로 개선된 3가지 결정적 이유

#### ① 2-Step OLS → Joint MLE (delta 과소추정 해결)

v8에서 delta를 OLS로 사후 추정하면 GARCH 잔차 노이즈 때문에 값이 과소추정됩니다.  
v9에서는 Hansen(1994) Skewed-t 우도 함수를 직접 설계하여  
`μ, ω, α, β, δ, ν, λ` 7개 파라미터를 동시 최적화(Joint MLE)합니다.

```
v8 delta (OLS):  0.021210 → 잔차 노이즈 포함
v9 delta (MLE):  0.000081 → 모든 파라미터와 동시 최적화된 순수 delta
```

#### ② GPR 고정 → AR(1) 감쇠 (비현실적 발산 해결)

v8에서 GPR을 42일 내내 고정하면 후반부 밴드가 비현실적으로 발산합니다.  
v9에서는 GPR이 시간에 따라 평시 수준(100)으로 점진 감쇠합니다.

```
GPR_t = 장기평균(100) + (GPR_0 - 100) × 0.92^t
→ 이란전쟁 GPR=705: D+21 시점 약 307로 감소
```

#### ③ 대칭 t분포 → 비대칭 Skewed-t (하방 비현실성 해결)

v8에서 Q10이 -50%까지 떨어지는 비현실적 결과가 나온 원인은  
대칭 t분포가 상방과 하방 꼬리를 동일하게 처리하기 때문입니다.  
v9에서는 λ(비대칭도) = -0.033을 학습하여  
**유가 하방 경직성**을 모델이 스스로 반영합니다.

```
λ = -0.033: 미세한 하방 편향
→ GPR 등 상승 요인이 제거된 후 순수 시장 잔차의 꼬리가 약간 하방
→ 해석: 모델이 과거 경제위기(하락장) 패턴까지 포착
```

> v9 고도화에서 Joint MLE, GPR 감쇠, Skewed-t 분포를 추가함으로써
